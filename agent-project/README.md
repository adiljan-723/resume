# 🎓 智能电子工程学习助手 Agent

基于 **Dify + RAG + 多工具调用** 的AI助教 Agent，能在面试中展示：**Agent设计能力、工作流编排、RAG全链路、Prompt工程**。

---

## 📁 项目文件说明

```
agent-project/
├── README.md              ← 本文件：搭建指南
├── dify-workflow.yml      ← Dify工作流配置（可导入）
├── test-cases.md          ← 测试用例
└── knowledge-base/
    └── 电子工程知识库.md   ← RAG知识库素材
```

## 🚀 搭建步骤（约30分钟）

### 第一步：注册 Dify Cloud（5分钟）

1. 打开 [cloud.dify.ai](https://cloud.dify.ai)
2. 用 GitHub 或 Google 账号注册/登录
3. 首次登录会提示创建 Workspace，名称为：`EE-Agent`
4. 进入后看到 Dify 主界面

> 💡 Dify Cloud 免费额度：每月 200 次 GPT-4o 调用，足够搭建和演示

---

### 第二步：创建知识库（10分钟）

1. 点击顶部导航 **「知识库」** → **「创建知识库」**
2. 填写：
   - 名称：`电子工程教材知识库`
   - 描述：`深圳大学电子信息工程专业核心课程知识`
3. 上传文件：将 `knowledge-base/电子工程知识库.md` 拖入
4. 分段设置：
   - 分段方式：`自动分段与清洗`
   - 索引方式：`高质量`
   - 检索方式：`混合检索`
5. 点击 **「保存并处理」**，等待 Embedding 完成（约1-2分钟）

---

### 第三步：导入工作流（5分钟）

1. 回到首页，点击 **「创建应用」** → **「Chatflow」**
2. 应用名称：`电子工程学习助手`
3. 进入编辑器后，点击左上角 **「导入」**
4. 将 `dify-workflow.yml` 文件内容粘贴进去
5. 导入后需要手动绑定知识库：
   - 点击 **「知识库检索」** 节点
   - 在右侧面板的 **「知识库」** 下拉中选择 `电子工程教材知识库`
6. 点击右上角 **「发布」**

---

### 第四步：配置模型（2分钟）

在 Dify 设置中配置模型供应商：

| 模型供应商 | 用途 | 获取 API Key |
|-----------|------|-------------|
| **OpenAI** | GPT-4o（专家链）/ GPT-4o-mini（分类+整合） | [platform.openai.com](https://platform.openai.com) |
| **DeepSeek**（备选） | 低成本替代方案 | [platform.deepseek.com](https://platform.deepseek.com) |

> 💰 成本预估：单次对话约 ¥0.02-0.08（GPT-4o-mini分类 + GPT-4o专家 + RAG检索）

---

### 第五步：测试验证（5分钟）

打开测试用例 `test-cases.md`，逐条测试：

1. **电路分析**："设计一个截止频率1kHz的低通RC滤波器"
2. **信号处理**："FFT频谱泄漏怎么用加窗解决？"
3. **单片机**："STM32配置外部中断的完整代码"
4. **兜底测试**："量子计算机什么时候替代经典计算机？"

观察 Dify 日志面板中的路由情况，确保：
- ✅ 意图分类正确
- ✅ 知识库检索有命中
- ✅ 专家链输出格式规范
- ✅ 回答包含可运行代码

## 🧠 Agent 架构详解（面试时讲这个）

```
用户输入
   │
   ▼
┌──────────────┐
│  意图识别      │  ← GPT-4o-mini 分类器
│ (5类电子工程   │     判断问题属于：电路/信号/单片机/通信/通用
│  问题分类)     │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  知识库检索    │  ← RAG (混合检索 + Rerank)
│  (向量DB +    │     从教材知识库中检索Top5相关片段
│   关键词混合)  │
└──────┬───────┘
       │
       ▼  (根据分类路由)
┌──────────────────────────────────────────┐
│           专家链 (并行执行)                │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐  │
│  │电路分析   │ │信号处理   │ │单片机开发 │  │
│  │GPT-4o    │ │GPT-4o    │ │GPT-4o    │  │
│  │MATLAB代码 │ │频域分析   │ │C代码     │  │
│  └──────────┘ └──────────┘ └──────────┘  │
│  ┌──────────┐ ┌──────────┐               │
│  │通信原理   │ │学习规划   │               │
│  │GPT-4o    │ │GPT-4o-mini│               │
│  │系统框图   │ │课程建议   │               │
│  └──────────┘ └──────────┘               │
└──────────────────────────────────────────┘
       │
       ▼
┌──────────────┐
│  回答整合     │  ← GPT-4o-mini
│  + 记忆存储   │     整合专家输出为结构化回答
└──────┬───────┘
       │
       ▼
   结构化回答输出
```

### Agent 四要素在本文中的体现：

| Agent要素 | 本文现 |
|-----------|--------|
| **感知 (Perception)** | 意图识别节点 — 理解用户想解决什么类型的问题 |
| **决策 (Planning)** | 条件路由 — 根据分类结果选择最合适的专家链 |
| **执行 (Action)** | 专家链 + 知识库检索 + 代码生成 — 多工具协调完成任务 |
| **记忆 (Memory)** | Dify 对话变量 — 记录上下文实现多轮对话 |

## 🔧 进阶扩展（选做）

如果面试官追问，可以说：

**Q: 如何评估这个Agent的效果？**
> 我设计了评测体系——意图识别准确率、知识库命中率、答案可用性评分三维度。每个维度有5个标准测试用例。

**Q: 如何进一步优化？**
> 下一步计划：加入 Memory 模块记录用户学习进度，实现个性化推荐。同时引入用户反馈机制形成数据飞轮。

**Q: 为什么选 Dify 而不是 LangChain？**
> Dify 可视化编排效率高，适合快速验证。如果需要更底层的控制（如自定义Agent调度策略），会迁移到LangChain。
